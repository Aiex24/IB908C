SAMMANFATTNING

Detta projekt är ett gesällprov i kursen Webbutveckling - Klientsidan, och är anpassat främst för att köras i Chrome.

Syftet med arbetet är att göra ett gesällprov. Kriterierna för gesällprovet är att det skall baseras på de tekniker som kursen bygger på och ska vara bra både till form och funktion.

Programmet har mestadels skrivits i Microsoft Visual Studio Community 2017. Koden har därefter flyttats ut till rena textfiler med hjälp av Notepad++ v7.5.7, för att få ett mer rent program för att begränsa sig till de tekniker som ingått i kursen.
För att kunna visa kod till exemplen på ett mer visuellt tilltalande sätt än att bara använda <code> utan att behöva skriva allt för mycket HTML-kod har ett script gjorts. Detta script lägger till CSS för att efterlikna hur koden ser ut i ett utvecklingsverktyg. Scriptet bygger mestadels på RegEx för att dela upp varje rad i delar med olika färg. För att lösa dessa RegEx-uttryck har en RegEx-tester används som går att hitta på https://regex101.com/

All HTML-kod och CSS har validerats med w3s validator som finns på https://validator.w3.org/

Resultatet är en hemsida som visar kursens uppgifter liknande ett dokument i A4-format med interaktiv funktion, och beskriver hur sidan är uppbyggd.

Detta projekt är ett gesällprov i kursen Webbutveckling - Klientsidan, och är anpassat främst för att köras i Chrome.

1.1 Bakgrund

IB908C Webbutveckling - Klientsidan 7,5 hp är en kurs på Institutionen för data- och systemvetenskap (DSV) Stockholms Universitet. Syftet med arbetet är att göra ett gesällprov. Kriterierna för gesällprovet är:

  • Ska baseras på de tekniker som kursen bygger på men får givetvis även innehålla tekniker från andra områden
  • Ska vara bra både till form och funktion
  • Ska inte vara en av de tidigare uppgifterna på kursen men får gärna vara en utökning av en eller flera av dessa

1.3 Mål

Målet med projektet är att skapa en hemsida som med HTML, CSS, och JavaScript efterliknar ett PDF-dokument med interaktiv funktion för att beskriva uppgifterna i kursen och sig självt.

Programmet har mestadels skrivits i Microsoft Visual Studio Community 2017 med MVC struktur och C# på backend-sidan. Koden har därefter flyttats ut till rena textfiler med hjälp av Notepad++ v7.5.7, för att få ett mer rent program för att begränsa sig till de tekniker som ingått i kursen och C#-koden har tagits bort.

Scriptet bygger mestadels på RegEx för att dela upp varje rad i delar beroende på vilken färg som skall användas. För att lösa dessa RegEx-uttryck har en RegEx-tester används som går att hitta på regex101.com

2.1.1 Grunderna i HTML5 (U1.1)

Doctype
Är en förkortning av "Document type decleration", och är en instruktion till webbläsaren vilket markup language som skall användas. För att använda sig av HTML5 används taggen <!DOCTYPE html>

Teckenkod
Teckenkod avgör hur t.ex en byte skall tolkas av webbläsaren. Själva värdet på byten ändras inte när den överförs mellan två enheter, men det är upp till teckenkodningen att avgöra hur värdet skall tolkas.

Språk
Programspråk är det sätt en människa använder för att skriva program med instruktioner som tolkas av datorns processor. En processor klarar bara av att tolka 1:or och 0:or, vilket är extremt tidskrävande att direkt skriva program med. I programspråket kan instruktionerna skrivas på en nivå som är betydligt mer mer intuitivt. Denna kod översätts sedan av ett annat program till maskinkod.

Validering
Precis som ett vanligt språk som har en uppsättning av regler gällande grammatik och vokabulär, så förväntas även datorprogram och webbsidor att följa de uppsatta reglerna som gäller för det språket eller ramverket. Att ett program är validerat är dock ingen garanti för att programmet är väl konstruerat.

Hur ett HTML-dokument är uppbyggt
Ett HTML-dokument börjar med definieringen vilket markup language (märkspråk) som skall användas. Exempelvis för att använda sig av HTML5 så inleds html dokumentet med <!DOCTYPE html> vilket i sig självt inte är en html-tagg. Sedan kommer så kallade taggar, vilka börjar med <tagg-namn> och har en avslutande del </tagg-namn>.

Den första taggen är <html> vilket utgör basen och berättar för webbläsaren att dokumentet är ett HTML-dokument. Det är inom html-taggarna som alla andra HTML-element ligger. Efter html-taggen kommer head-taggen som är en behållare för alla huvudelementen. Exempelvis: <title>, <style>, <script>, <noscript>, <link>, <base>, <meta>. Därefter kommer body-taggen som innehåller sådant som syns i själva webbläsaren, som till exempel bilder, tabeller, text med mera.

2.1.2 Semantiska element för struktur (U1.2)

<article>
Omsluter en sammansättning som på egen hand kan återges utan att tappa innebörden. article-taggen kan innehålla andra article-taggar och section-taggar

<nav>
Används för att omsluta en grupp av navigationslänkar. Skillnaden mot en <div> är att nav-taggen berättar för programvara som hjälper funktionsnedsatta att detta område innehåller navigationslänkar.

<header>
Läggs inuti article-taggarna och omsluter inledande innehåll. Exempel på detta är h-taggar eller information om webbsidans utvecklare

<footer>
Läggs inuti article-taggarna och innehåller vanligtvis information om webbsidans utvecklare, äganderätt, kontaktinformation eller länk till toppen av sidan.

<section>
Innehåller ofta en header men är inget måste. Tumkravet för att använda en section-tagg brukar vara att om kraven för att använda en article-tagg ej är uppfyllda, men sektionen med dess header skulle bli en del av innehållsförteckningen så kan man använda section-taggen. section-taggen kan nästlas inuti en article-tagg eller läggas som en fristående del. <figure>
Används för att bland annat visa bilder, grafer med mera. figure-taggen är en del av huvudinnehållet men kan flyttas till exempelvis en bilaga utan att påverka flödet av huvudinnehållet.

Skapar en Window gadget som består av en togglingsknapp som expanderar eller minimerar ett fält där ytterligare innehåll kan läggas.

<summary>
Läggs först inom details-taggarna, och kommer visas som rubrik på togglingsknappen i både stängt och öppet läge. Om ingen summary-tagg finns med, sätter webbläsaren själv ett defaultvärde (för Chrome är defaultvärdet Info).

2.2 Design med CSS 3

2.2.1 Grunderna i CSS 3 (U2.1)

Externt länkad CSS-fil och dess fördelar
Det finns tre sätt att implementera CSS: External Style Sheet, Internal Style Sheet ovh Inline Style. Fördelen med den förstnämda är att man kan använda samma fil för att ändra utseende på alla sidor som använder filen samtidigt, istället för att ändra varje sida var för sig. Detta underlättar då man vill ha ett enhetligt utseende på ett projekt med flera sidor.

Arv
Arv läggs till på en property för att sätta värdet till samma värde som elementet det ligger i har. Default värdet på olika properties varierar.

/*background: Före: transparent. Efter: white*/ background: white; /*color: Före: Samma som parent. Efter: blue*/ color: blue; /*margin: Före: 0 Efter: Samma som parent*/ margin: inherit;

Cascade
Cascade är en algoritm som avgör vilken CSS som skall användas på en property då flera värden har blivit angivna.

Selektorer (tagg-, attribut-, klass-, id- och barn-selektorer)
Det finns olika sätt att definiera en selektor. Nedan återfinns några av dem.

/*Tagg: Sätter CSS på alla h1 element*/ h1 { color: red; } /*Attribut: Sätter CSS på alla element som har attributet target=su*/ [target=su] { background: red; } /*Klass: Sätter CSS på alla element med klass-namnet "myClass"*/ .myClass { color: red; } /*id: Sätter CSS på elementet med id=myId*/ #myId { color: red; } /*barnelement: Sätter CSS på alla h1 som har en header som parent*/ header > h1{ color: green; } /*barnelement: Sätter CSS på alla h1 inuti en header*/ header h1 { color: green; }

Enheter (absoluta: in, cm, mm, pt, pc, px och relativa: em, ex, %)
Enheter kan delas in absoluta och relativa storlekar. De absoluta behåller samma storlek medan de relativa beror på hur stort browserfönstret är eller storleken på dess parent-element.
