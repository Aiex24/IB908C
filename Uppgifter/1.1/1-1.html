1.1 Grunderna i HTML5

Doctype
Är en förkortning av "Document type decleration", och är en instruktion till webbläsaren vilket markup language som skall användas. För att använda sig av HTML5 används taggen <!DOCTYPE html>

Teckenkod
Teckenkod avgör hur t.ex en byte skall tolkas av webbläsaren. Själva värdet på byten ändras inte när den överförs mellan två enheter, men det är upp till teckenkodningen att avgöra hur värdet skall tolkas.

Språk
Programspråk är det sätt en människa använder för att skriva program med instruktioner som tolkas av datorns processor. En processor klarar bara av att tolka 1:or och 0:or, vilket är extremt tidskrävande att direkt skriva program med. I programspråket kan instruktionerna skrivas på en nivå som är betydligt mer mer intuitivt. Denna kod översätts sedan av ett annat program till maskinkod.

Validering
Precis som ett vanligt språk som har en uppsättning av regler gällande grammatik och vokabulär, så förväntas även datorprogram och webbsidor att följa de uppsatta reglerna som gäller för det språket eller ramverket. Att ett program är validerat är dock ingen garanti för att programmet är väl konstruerat.

Hur ett HTML-dokument är uppbyggt
Ett HTML-dokument börjar med definieringen vilket markup language (märkspråk) som skall användas. Exempelvis för att använda sig av HTML5 så inleds html dokumentet med <!DOCTYPE html> vilket i sig självt inte är en html-tagg. Sedan kommer så kallade taggar, vilka börjar med <tagg-namn > och har en avslutande del </tagg-namn>.

Den första taggen är <html> vilket utgör basen och berättar för webbläsaren att dokumentet är ett HTML-dokument. Det är inom html-taggarna som alla andra HTML-element ligger. Efter html-taggen kommer head-taggen som är en behållare för alla huvudelementen. Exempelvis: <title>, <style>, <script>, <noscript>, <link>, <base>, <meta>. Därefter kommer body-taggen som innehåller sådant som syns i själva webbläsaren, som till exempel bilder, tabeller, text med mera.


Struktur-taggar

<body>
Innehåller sådant som syns i själva webbläsaren, som till exempel bilder, tabeller, text med mera.

<head>
Är en behållare för alla huvudelementen. Vilka är: <title>, <style>, <script>, <noscript>, <link>, <base>, <meta>, där endast den första måste vara med.

<html>
Utgör basen och berättar för webbläsaren att dokumentet är ett HTML-dokument. Det är inom html-taggarna som alla andra HTML-element ligger.

<title>
Dokumentets titel. Det är denna text som syns på fliken i webbläsare, som titel vid sökning, och som default-titel om sidan sparas som favorit.


Några av de attribut som kan användas med meta-taggen är:

author
Här kan man skriva in den person eller organisation som har gjort webbsidan. Till skillnad från title, keywords, och description, så påverkar inte "author" sökresultatet.

description
Här beskrivs kortfattat webbsidans innehåll. De flesta sökmotorer använder denna information för att få fram sökresultat. Varje webbsida i ett projekt bör ha sin egen beskrivning.

generator
Här listas den programvara som använts för att skapa webbsidan

keywords
Här listas de nyckelord som man vill att ens webbsida ska hittas av. Om nyckelordet även återfinns i titeln och sidans innehåll så rankas dessa generellt högre av sökmotorn.

robots
Berättar för webbläsaren hur webbsidan ska indexeras. Man kan välja att indexera första sidan ("index/noindex") och restrerande (follow/nofollow"), och alla kombinationer av dessa två. Alternativt kan index och follow ersättas av "all" för att få att alla sidor indexeras.

http-equiv
Med http-equiv kan man bland annat sätta expire datum för webbsidan, välja att inget sparas till användarens cache, välja vilket värde bilderna ska ha på ens hemsida som till exempel "barntillåtet"


Text-taggar

<abbr>
Används för akronym för att lägga till ett tooltip med fullständig betydelse. Exempelvis de tre komponenter som ingår i denna kurs. HTML, CSS och JS.

<address>
I adress-taggen läggs kontaktinformation (adress, email, eller någon annan form av kontaktinformation) och kan läggas antingen direkt i <body>, <article> eller <footer>, med relevant information till det element det ligger i.

<blockquote>
Används då man vill använda sig av ett längre citat. Webbläsare lägger vanligtvis till ett indrag på citatet.

Stockholms högskola grundades 1878 med ambitionen att förnya de traditionella universiteten i Sverige. Nära kontakter och ett aktivt erfarenhets- och kunskapsutbyte med omvärlden präglar verksamheten sedan starten.

<br>
br-taggen är en så kallad singleton tagg (kräver ingen stängningstagg) och används för radbrytning.

<cite>
Används för titlar för exempelvis böcker filmer artikler, källhänvisning med mera.

<code>
Stöds av de flesta webbläsare och sätter typsnittet på en text för att efterlikna datorkod, utan att man behöver använda CSS. För att använde taggen skriver man:

<code>Text</code>

<dfn>
Används för att markera det som definieras i en mening eller fras. Man kan välja att lägga till en title tagg med eller utan abbr-tagg inuti dfn-taggen

Den här texten ligger inom dfn-taggar

<div>
Används för att gruppera element, och/ tillsammans med CSS för att styla en webbsida.

<em>
Används för att betona (Emphasize) en text.

<h1>,<h2>,<h3>,<h4>, <h5>,<h6>
Används för att definiera rubriker. <h1> används för huvudrubrik och de övriga som underrubriker.

<kbd>
Används för att beskriva tangentbordskommandon.

<p>
Definierar en paragraf.

<pre>
Till skillnad mot <p> så behåller <pre> formattering som exempelvis flera mellanslag i rad.

<q>
Används vid kortare citat

<samp>
Används för att presentera resultatet av ett datorprogram.

<strong>
Används för att markera att ett ord är viktigt. Om inte CSS används renderas <strong> och <b> (bold) oftast lika av webbläsaren.

<var>
Definierar att texten inom taggen är en variabel.

<a>
Används för att skapa hyperlänkar. På mozilla.org kan du läsa mer om a-taggen.

<dl>, <dt> och <dd>
Används tillsammans för att skapa en definiationslista. Nedanstående lista är gjord med dessa taggar

<dt></dt>

<dd></dd>

<li>
Definierar en komponent i en lista

<ol>
Om li-komponenten sätts inom ol-taggar (ordered list) så numreras dessa.

  1. Element A
  2. Element B
  3. Element C

<ul>
Om li-komponenten sätts inom ul-taggar (unordered list) så dekoreras dessa med listpunkter

  • Element A
  • Element B
  • Element C
