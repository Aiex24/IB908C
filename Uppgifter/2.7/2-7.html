2.7 Sidlayout

Webbsideslayout
fixed: Innebär att man ger storleken ett statiskt värde som inte beror på storleken på fönstret. fluid: Innebär att storleken inte har ett statiskt värde. De kan antingen bero på storleken på behållaren de ligger i, användarens inställningar, eller de komponenter de innehåller. elastic: Innebär att elementets storlek sätts i em som är proportionell mot teckenstorleken.
Anledningen till att använda CSS istället för tables vid webbsideslayout är att man får mer kontroll och flexibilitet med CSS. Man slipper lägga massa kod i html-en och får därmed sidor som laddas snabbare. Man kan återanvända CSS kod vilket ger kortare kod [CSS1].

Position
Några av de värden positionen kan sättas till är:
absolute - positionen sätts relativt till närmaste förfader element som inte har värdet static
fixed - positionen sätts relativt webbläsaren, och påverkar inte det normala flödet
relative - elementet positioneras relativt det normala flödet
width - sätter bredden på ett element
height - sätter höjden på ett element
max-width - största bredden ett element kan sättas till
max-height - största höjden ett element kan sättas till
min-width - minsta bredden ett element kan sättas till
min-height - minsta höjden ett element kan sättas till.

Även om offset värden (top, bottom, left, right) adderas, kommer detta ej påverka andra elements position
static - elementet renderas i den ordning de kommer i koden och fyller på flödet
